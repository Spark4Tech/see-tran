See-Tran: Agent-assisted crowdsourcing guide

See-Tran is an open-source web app for modeling a transit agency’s technology landscape and benchmarking across agencies.

Core data model

  • Agency – a public transit organization
  • FunctionalArea → Function (with Criticality)
  • Component – system/subsystem used by agencies
  • Configuration – Agency + Function + Component (+ Status, Deployment Date, Version Label, Notes)
  • Vendor, Product, ProductVersion (linked via ConfigurationProduct)
  • IntegrationPoint, Standard (early stage)
  • Tags/TagGroups (optional)

How contributors use the app

  1. Browse & search (print-friendly views for functions & functional areas)
  2. Create a Configuration (filters, quick create, wizard)
  3. Keep entities tidy (concise names; avoid duplicates; vendor nomenclature)
  4. Print and share: /functional-areas/print, /functions/print

Agents (app/agents/)

  • agency_agent.py, vendor_agent.py, component_agent.py
  • Suggest mode with provenance; reviewer UI planned
  • Data quality: verifiable sources, official sites, mark uncertainty, normalize names

Architecture

  • Backend: Flask + SQLAlchemy + Alembic
  • Frontend: Jinja + Tailwind + HTMX
  • Auth: login_required; OAuth for hosted builds
  • Data: /data, loaders in /scripts

Quick start: Python 3.12+, pip install -r requirements.txt, flask run.
